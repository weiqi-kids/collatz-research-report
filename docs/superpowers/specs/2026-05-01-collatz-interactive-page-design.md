# Collatz 猜想互動研究報告 — 設計規格

> 日期：2026-05-01
> 類型：一頁式互動網頁（純靜態單一 HTML 檔）
> 技術棧：D3.js + MathJax + CSS Custom Properties (OKLCH)
> 目標受眾：高中生（白話引言）+ 研究者（完整數學推導）

## 1. 設計決策摘要

| 決策項 | 選擇 | 理由 |
|--------|------|------|
| 設計方案 | 敘事引導式（方案 A） | 最符合研究報告定位，敘事性強 |
| 導覽方式 | 捲軸為主 + 頂部黏著導覽條 | 頂部橫向章節，捲動時 active 狀態切換 |
| 數學公式 | MathJax + 圖解文字輔助 | 完整 LaTeX 渲染，白話文補充直覺 |
| 語言 | 中英混合 | 標題和說明繁體中文，數學術語保留英文 |
| 部署方式 | 純靜態單一 HTML 檔 | CSS/JS 內嵌，D3.js 和 MathJax 從 CDN 載入 |
| 配色系統 | design-tokens skill（OKLCH light theme） | 見 §3 |
| 內容範圍 | 全部 7 個框架 + 所有章節完整保留 | 不刪減不簡化 |
| 互動圖數量 | 7 個主框架各 ≥ 2 張 + 輔助章節各 1 張，共 20 個互動元件 | 見 §5 |
| Collatz 映射 | 統一使用壓縮映射 `S(n) = (3n+1) / 2^{v₂(3n+1)}`，所有圖表（含英雄區）均使用此定義 | 與研究報告一致 |

## 2. 頁面骨架

### 2.1 頂部黏著導覽條

- 固定於視窗頂部，`position: sticky; top: 0; z-index: 100`
- 背景 `--bg-surface` + 底部 `--border-subtle` 分隔線
- 水平排列章節縮寫（可橫向捲動）：

  `問題` `Epoch` `轉換器` `連分數` `序數` `反向樹` `ℤ₂` `K複雜度` `實驗` `化約` `有界` `障礙` `結論`

- 捲動時用 `IntersectionObserver` 偵測當前段落，active 項目底部加 `--color-link` 的 3px 指示線
- 字級 `--text-xs` (18px)
- 頂部閱讀進度條：細線（2px），`--color-link` 色，寬度 = 捲動百分比

**響應式（≤ 768px）**：右上角漢堡按鈕（三橫線圖示，`--text-primary` 色），點擊展開全寬下拉面板（`--bg-surface` 背景，`box-shadow: 0 4px 12px oklch(0 0 0 / 0.1)`）。展開動畫 `max-height` 過渡 300ms ease。面板外點擊或選擇章節後自動收合。閱讀進度條保留於導覽條下方。

### 2.2 英雄區（Hero）

- 報告標題 `--text-3xl` (56px)
- 副標題（日期、性質、結論）`--text-base` (24px)
- 結論「未解決」用 `--color-medium` badge 標記，badge 字級 `--text-sm` (20px) + bold（滿足 WCAG 大字門檻 ≥ 18.66px bold）
- **快速體驗互動**：
  - 大型輸入框（視覺焦點），輸入任意正整數
  - 即時繪製 Collatz 軌道折線圖（D3 line chart）
  - x 軸 = 壓縮映射步數，y 軸 = 值，可切換線性/對數
  - 逐步動畫播放模式（點擊「播放」）
  - 「深入分析」按鈕跳轉至框架 1
- **全域 n 輸入機制**：此 n 同時傳遞給所有框架圖（1A, 1B, 2A, 4A, 6B, 7A, 7B）同步更新
- **輸入驗證**：僅接受 1 ~ 10⁹ 的正整數。非法輸入時輸入框邊框變為 `--color-critical` 並顯示內嵌提示「請輸入 1 至 1,000,000,000 的正整數」。超出範圍時提示「超出上限，已截斷至 10⁹」。頁面載入時預設 n = 27（經典示範值，壓縮映射軌道長度 41 步）。

### 2.3 章節容器

- 最大寬度 72rem（`max-width: 72rem`），水平置中
- **主章節**（H2 級）間距 `4rem`（本報告內容密度高，需更大呼吸空間）
- **章節內子區塊**間距 `2rem`（遵循 output-rules.md §B）
- 章節間用淡色分隔線 `--border-subtle` 區隔
- 所有 `.chart-container` 設定 `position: relative; z-index: 1`，確保不高於導覽條。圖表內 tooltip/popover 使用 `z-index: 50`。
- 每個章節統一結構：

  ```
  H2 標題 (--text-xl, 32px)
  → 白話引言 (--text-base, 24px)
  → 互動圖 A
  → 數學推導（MathJax）
  → 互動圖 B
  ```

- 七個主框架對應頁面章節 §2-§8，在 §5 中以「框架 1」~「框架 7」標示。對應關係：框架 1 = §2, 框架 2 = §3, ..., 框架 7 = §8。

### 2.4 斷裂點 Callout

## 4. 技術架構

### 4.1 CDN 依賴

CDN 腳本放在 `<head>` 中使用 `defer`：

```html
<head>
  <!-- D3.js v7 -->
  <script src="https://d3js.org/d3.v7.min.js" defer></script>

  <!-- MathJax 3 -->
  <script src="https://cdn.jsdelivr.net/npm/mathjax@3/es5/tex-mml-chtml.js" defer></script>
</head>
```

**MathJax 配置**：行內公式用 `\( ... \)`，區塊公式用 `\[ ... \]`（避免 `$` 與 JavaScript 衝突）。D3 動態插入含 LaTeX 的 DOM 後，呼叫 `MathJax.typesetPromise([element])` 重新渲染。配置 `MathJax.startup.pageReady` 延遲至首屏公式可見時才觸發。

### 4.2 效能策略

| 策略 | 實作方式 |
|------|---------|
| 懶載入 | `IntersectionObserver` 偵測章節進入視窗才初始化 D3 圖表，離開時銷毀（5A/5B 連動例外，見 §5） |
| MathJax 延遲渲染 | 先渲染可見區域公式，捲動時再渲染下方 |
| 重計算隔離 | 計算實驗 playground 使用 Web Worker 避免 UI 凍結 |
| Canvas fallback | k > 14 的散佈圖（3B）用 canvas 渲染 |
| 預計算資料 | k ≤ 18 的完整枚舉資料嵌入 HTML（JSON），避免即時計算 |
| 輸入限制 | 軌道圖 n ≤ 10⁹，熱力圖 n ≤ 10⁶ |
| CDN 降級 | D3/MathJax CDN 失敗時，圖表區域顯示靜態截圖預覽 + 「互動功能無法載入」提示；公式顯示純文字 ASCII fallback |
| CSP 相容 | 頁面 `<meta>` 中設定 `Content-Security-Policy` 允許 `worker-src blob:` |

### 4.3 全域 n 輸入架構

```
英雄區輸入框
    ↓ 發佈事件 collatz:n-changed
    ↓
  [1A] [1B] [2A] [4A] [6B] [7A] [7B]
  各圖監聽事件，自動更新
```

每個圖也可獨立輸入覆蓋全域值。

**覆蓋規則**：獨立輸入後，該圖進入「脫鉤」狀態，輸入框右側顯示鏈結解鎖圖示（`--text-muted` 色）；點擊圖示或下次全域 n 變更時自動重新同步。脫鉤狀態下輸入框邊框改為 `--border-subtle` 虛線以示區別。

單一 HTML 檔內部組織：

### 框架 1：Epoch 分解與位移座標

#### 圖 1A：軌道動畫

| 屬性 | 規格 |
|------|------|
| 類型 | D3 折線圖 + 步驟標記 |
| 輸入 | n（slider + 輸入框），接收全域 n |
| 互動 | 逐步播放（含播放速度控制）；「跳到下一 epoch」按鈕 |
| 標記 | 每步標示 `v₂ = k`（壓縮映射），epoch 用交替背景色區分 |
| y 軸 | 可切換線性/對數；對數模式疊加 `log(3/2)` 增長率和 `-log 2` 下降率參考線 |
| 色彩 | epoch 背景色用 OKLCH 固定 L=0.94, C=0.02，旋轉 H 生成 |
| 限制 | n ≤ 10⁹ 或軌道 ≤ 2000 步 |
| 邊界 | n = 1 時顯示單點圖 + 提示「1 是 Collatz 猜想的終點，軌道長度為 0」。n = 2 時顯示一步軌道 (2→1)。 |
| 動畫規格 | 預設間隔 500ms/步，速度切換 0.5x / 1x / 2x / 4x（按鈕組）。軌道超過 200 步時自動切換為 4x 並提示「軌道較長，已加速播放」。提供「跳到結束」按鈕。 |

#### 圖 1B：Epoch 瀑布圖

| 屬性 | 規格 |
|------|------|
| 類型 | 瀑布圖（waterfall chart） |
| 輸入 | 同 1A 的 n |
| 結構 | 橫軸每格一個 epoch。`t_i` = epoch i 的奇數步（×3+1）次數，`w_i` = 額外除 2 次數（即該 epoch 總 v₂ 減去 `t_i`）。每格高度 = 淨對數效果 `t_i · log₂(3/2) - w_i`。 |
| 色彩 | 正（增長）= `--color-critical` 紅 + ▲，負（下降）= `--color-low` 藍 + ▼ |
| 附加 | 累計線顯示軌道對數值變化；零線標示 epoch 淨效果平衡點（淨效果 = 0）；圖旁注釋「當 epoch 的 v₂ 平均值 > log₂3 ≈ 1.585 時，該 epoch 使數值縮小」 |
| 白話 | 圖旁標註「紅色 ▲ = 爬坡，藍色 ▼ = 下坡，藍色總面積比紅色大就代表數字在變小」 |

### 框架 2：3 態轉換器與譜間隙

#### 圖 2A：狀態機動畫

| 屬性 | 規格 |
|------|------|
| 類型 | 固定三角形佈局狀態機（3 節點 A/B/C）+ bit 流 |

| 動畫 | 逐位播放 carry 傳播；當前狀態節點高亮；當前轉移弧線動畫 |
| 輸出 | 輸入 bit 流下方即時顯示輸出 bit 流 |
| 統計 | 節點大小隨停留次數膨脹 + 旁顯經驗停留比例（如「A: 34%」） |
| 附加 | v₂ 計數器：即時顯示連續 0 輸出位計數 |

#### 圖 2B：譜收斂圖

| 屬性 | 規格 |
|------|------|
| 類型 | 三條折線圖 + 距穩態距離副圖 |

### 框架 4：序數值勢函數

### 輔助章節 B：有界軌道分析（對應頁面 §11）

#### 圖 B-1：循環方程視覺化

| 屬性 | 規格 |
|------|------|
| 類型 | 柱狀圖 |
| 互動 | slider 調 P（循環長度，1 ~ 50） |
| x 軸 | P（循環長度） |
| y 軸 | 對數刻度，顯示 P 步循環的最小可能元素 `m_min`。循環方程：若存在 P 步循環，其最小元素 m 滿足 `m = (Σᵢ 3^{P-i} · 2^{sᵢ}) / (2^B - 3^P)`，其中 B = 循環的總 v₂、sᵢ 為部分和。`m_min = ⌈3^P / (2^B - 3^P)⌉`，取 `B = ⌈P · log₂3⌉`（使分母 `2^B - 3^P` 為正的最小 B 值，此時分母很小，m_min 顯著增長）。 |
